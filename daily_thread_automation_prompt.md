# Daily Thread Automation Prompt

毎朝8時にこのThreadへ戻り、以下を順番に実行してください。

## 目的

昨日から今朝までのAI業界アップデートを収集し、過去2週間のこのThreadの発信履歴と被らない切り口で、非エンジニア向けの発信ドラフトを作る。最終結果はTriageで確認しやすい形にまとめる。

## 実行手順

1. このThreadの直近2週間の会話、出力、ユーザーフィードバックを読む。
2. `thread_seed_prompt.md` 相当の初期設定、発信ターゲット、トーン、避けたい表現、過去テーマを再確認する。
3. Web検索で、昨日から今朝までのAI業界アップデートを収集する。
4. 可能ならNotion、Slack、Gmail、Microsoft Suiteなどの接続済みプラグインから素材バンクを読む。
5. 収集したニュースと素材をもとに、過去2週間と被らない切り口を3つ提案する。
6. 3つの中から、発信者自身の体験、失敗談、試行錯誤と紐付けやすい切り口を1つ選ぶ。
7. 選んだ切り口で、X投稿用ドラフトを2案、note/ブログ用の長文骨子を1案作る。
8. 最後にTriageへ届けるための短い確認サマリーを作る。

## Web検索ルール

- 対象期間は「昨日 00:00 から今日 08:00 まで」を基本にする。
- 重要ニュースが少ない場合のみ、直近72時間まで広げてよい。
- 公式発表、一次情報、信頼できる報道を優先する。
- 出典名、公開日、何が新しいのかを明記する。
- 未確認情報は「未確認」「噂」「観測」と明示し、ドラフトの主軸にはしない。
- AIモデル、プロダクト、規制、資金調達、企業提携、研究、開発者向け発表のうち、非エンジニアの行動に影響しそうなものを優先する。

## 素材バンク参照ルール

接続済みプラグインがある場合は、以下を探す。

- Notion: ネタバンク、過去の投稿アーカイブ、発信メモ
- Slack: 自分が保存したAI関連メモ、チーム内で反応があったAIニュース
- Gmail: AI系ニュースレター、イベント案内、プロダクトアップデート
- Microsoft Suite: OneNote、Word、Excel、PowerPoint内の発信メモや実績記録

プラグインが使えない場合は、このThreadの履歴と手元にある `material_bank_sources.md` 相当のメモだけで進める。

## 切り口の選び方

切り口3案は、必ず以下を含める。

- ニュース起点の要点
- 非エンジニアにとっての意味
- 過去2週間の投稿とどう違うか
- 自分の体験や失敗談に接続できる余地
- 想定読者の行動

3案のうち1案を選ぶときは、単にニュース性が強いものではなく、「自分の経験として語れるもの」を優先する。

## フィードバック反映ルール

ユーザーがこのThreadに返したコメントは、翌日以降の生成条件として扱う。

例:

- `2案目の方向で、もう少し失敗談を入れて`
  - 翌日以降は、切り口選定時に「失敗談へ接続しやすいか」を重く見る。
- `最近Anthropic系のネタが多いから今週はOpenAI/Google系に寄せて`
  - 今週中は企業配分をOpenAI/Google寄りに調整する。ただし重大ニュースがある場合は例外として扱う。
- `noteは実務寄りに厚くして`
  - note/ブログ骨子では、導入よりも実務ステップと判断基準を厚くする。

## 品質基準

- 非エンジニアが読んで意味がわかる。
- ニュースの翻訳で終わらず、仕事や行動への接続がある。
- 煽りではなく、判断材料になっている。
- 直近の発信履歴と切り口が重複していない。
- ユーザーのフィードバックが具体的に反映されている。
