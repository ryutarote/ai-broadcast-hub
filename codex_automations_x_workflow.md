# Codex Automations X投稿ワークフロー

Codex Automationsの活用方法を継続的に調査し、X用の投稿ドラフトへ落とし込むための運用フロー。

## 目的

Codex Automationsを「便利そう」で終わらせず、非エンジニアにも伝わる実務例に翻訳してX投稿にする。

重点は以下。

- Codex Automationsで何が自動化できるか
- どんな仕事を任せると効果が出やすいか
- どこに人間の判断やレビューを残すべきか
- 自分の発信運用にもどう使えるか

## 参照する公式情報

- OpenAI Academy: Codex Automations
- OpenAI Academy: What is Codex?
- OpenAI Codex製品ページ
- OpenAI Help Center: Using Codex with your ChatGPT plan
- OpenAI公式ブログ: Introducing the Codex app

公式情報で確認する論点。

- AutomationsはスケジュールやトリガーでCodexに定期作業を任せる仕組み
- Thread automationは同じ会話に戻り、既存文脈を使って作業を続けられる
- 良いautomationは具体的、反復可能、レビューしやすい
- ローカル実行ではCodexと端末が動いている状態のほうが安定しやすい
- Codexはコード以外にも、複数ソースの情報収集、ファイル更新、資料作成、定期レポートなどに使える

## 日次フロー

1. 前日から当日朝までのOpenAI/Codex関連の公式更新を確認する。
2. Codex Automationsの使い方に関係する新情報だけを抽出する。
3. 公式情報以外のニュースやユーザー投稿は、補助情報として扱う。
4. 非エンジニア向けに「何ができるか」ではなく「何を任せるとよいか」へ翻訳する。
5. X投稿の切り口を3案作る。
6. 3案のうち、体験談や運用改善に接続しやすい1案を選ぶ。
7. X投稿ドラフトを2案作る。
8. 投稿前レビュー用に、根拠、リスク、修正ポイントを短くまとめる。

## X投稿の品質基準

- 1投稿1メッセージに絞る
- 煽り語を使わない
- Codex Automationsを万能化しない
- 「任せる作業」と「人が判断する作業」を分ける
- 非エンジニアでも自分の業務に置き換えられる
- 公式情報にない断定をしない

## 投稿候補の型

```text
Codex Automationsの良さは「AIが勝手に全部やる」ではなく、
毎回やる定型作業を、レビュー前提で先に進めておけること。

たとえば毎朝、
・AIニュースを集める
・発信の切り口を3つ出す
・X投稿案を2本作る
・判断が必要な点だけ残す

ここまで自動化して、人間は最後の編集に集中する。
```

## 運用メモ

- 毎日動かす場合は、朝の発信準備前に出力される時刻にする。
- 週次で振り返り、反応がよい型を `daily_thread_automation_prompt.md` に戻す。
- 投稿実績、反応、採用した下書きは `material_bank_sources.md` の手動素材メモに追記する。
