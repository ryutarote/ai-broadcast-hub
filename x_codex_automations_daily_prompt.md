# X Codex Automations Daily Prompt

毎朝、Codex Automationsの活用方法を調査し、X用投稿ドラフトを作成してください。

## 目的

Codex Automationsを使った実務ワークフローを、非エンジニアにも伝わるX投稿に翻訳する。単なる機能紹介ではなく、「何を任せるとよいか」「どこを人間がレビューするか」「自分の仕事や発信運用にどう使えるか」まで落とし込む。

## 対象読者

- AI活用に関心がある非エンジニア
- 事業側、企画職、マーケター、経営者、個人クリエイター
- CodexやAutomationsに興味はあるが、開発者向けの説明だけでは使いどころを掴みにくい人

## 調査手順

1. OpenAI公式情報を優先して、Codex Automationsに関係する最新情報を確認する。
2. 公式情報で確認できない内容は、断定しない。
3. 公式情報以外のニュース、SNS投稿、ブログは補助情報として扱い、主張の根拠にはしない。
4. 前回までのこのThreadの出力、ユーザーフィードバック、採用/不採用の傾向を読む。
5. 過去2週間と同じ切り口を避ける。
6. 今日の投稿切り口を3案作る。
7. 3案から1案を選び、X投稿ドラフトを2本作る。
8. 投稿前の確認点を短くまとめる。

## 公式情報で見る観点

- Automationsで定期的に任せられる作業例
- Thread automationとStandalone automationの使い分け
- 同じ会話の文脈を使い続けることの利点
- レビューしやすい成果物の作り方
- ローカル実行時の注意点
- Codexをコード以外の情報収集、資料作成、ファイル更新、定期レポートに使う考え方

## X投稿で避ける表現

- AIが全部やってくれる
- 放置で稼げる
- 乗り遅れるな
- 革命、激変、衝撃、完全終了
- エンジニアだけに通じる専門用語だけで押し切る
- 公式情報にない機能や制限を断定する
