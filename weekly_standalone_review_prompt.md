# Weekly Standalone Automation Prompt

週1回、このプロンプトを別ThreadまたはStandalone automationで実行してください。

## 目的

日次ドラフト生成Threadとは役割を分け、過去1週間の発信パフォーマンスをレビューし、次週の発信方針を提案する。

## 入力

可能な範囲で以下を参照する。

- Xの投稿実績
- note/ブログの閲覧数、スキ、コメント、滞在時間
- Notionの過去投稿アーカイブ
- Slack/Gmail/Microsoft Suiteに残した反応メモ
- 日次Thread automationの出力履歴
- ユーザーが手動で追記した振り返り

## 実行手順

1. 過去7日間の投稿を一覧化する。
2. テーマ、企業、媒体、切り口、体験談の有無で分類する。
3. 反応が良かった投稿と悪かった投稿を比較する。
4. 数字がない場合は、保存、返信、引用、滞在、体感反応などの代理指標を使う。
5. 次週の企業配分、テーマ配分、投稿フォーマットを提案する。
6. 日次Thread automationへ戻すべき学習メモを作る。

## 出力形式

### 1. 今週の発信一覧

- 投稿日:
- 媒体:
- テーマ:
- 企業/モデル:
- 切り口:
- 体験談の有無:
- 反応:

### 2. パフォーマンス所見

- 良かった傾向:
- 弱かった傾向:
- 読者が反応した問い:
- 読者が反応しなかった表現:

### 3. 偏りチェック

- 企業の偏り:
- テーマの偏り:
- 媒体の偏り:
- トーンの偏り:

### 4. 次週方針

- 寄せる企業/領域:
- 減らす企業/領域:
- 増やす切り口:
- 減らす切り口:
- 失敗談/体験談の使い方:
- note/ブログで深掘りするテーマ:

### 5. 日次Threadへ戻す学習メモ

以下を日次Threadへ貼り戻せる形で出力する。

```text
次週の発信方針:
- 

日次ドラフト生成時に強めること:
- 

避けること:
- 

企業/テーマ配分:
- 
```
